## 📖 ส่วนที่ 1: คู่มือการใช้งานระบบ (User Manual)

ระบบ **MOTORBOY TOKYO OVERRIDE** เป็นเครื่องมือจำลองวิเคราะห์ข้อมูล CAN-BUS และปรับแต่งจูนกล่องเครื่องยนต์รถจักรยานยนต์ในสไตล์ Cyberpunk/Tokyo Underground ซึ่งมีทั้งหมด 4 โมดูลหลัก ดังนี้:

### 1. [01] Dyno Test Stage (แท่นทดสอบวัดแรงม้าและแรงบิด)
แท่นทดสอบไดโน่ใช้สำหรับการปรับแต่งอัตราการจ่ายน้ำมัน ไฟจุดระเบิด และตรวจสอบประสิทธิภาพของรอบเครื่องยนต์:
* **ECU Adjustments (ปรับจูนกล่อง)**: เลื่อนปรับค่า **Fuel Trim (อัตราการจ่ายน้ำมัน %)**, **Ignition Timing (องศาไฟจุดระเบิด BTDC)** และ **Rev Limit (รอบเครื่องสูงสุด RPM)**
* **Presest System (สูตรจูนสำเร็จรูป)**: มีแผนการจูนมาตฐานให้เลือก 4 แบบ เช่น **STOCK ECU** (ดั้งเดิม), **TOKYO SPORT** (เน้นสปอร์ต), **SHUTOKO DRAG** (ลากรอบสูง) และ **STEALTH ECO** (เน้นประหยัด)
* **Custom Maps (บันทึกสูตรจูนเอง)**: สามารถพิมพ์ชื่อสูตรที่ตนเองจูนและกด **"SAVE MAP"** เพื่อเก็บข้อมูลลงใน Local Storage และกดสลับใช้งานได้ทันที รวมถึงปุ่มลบสูตรที่ไม่ใช้งาน
* **Run Dyno Sweep (เริ่มวิ่งไดโน่)**: กดเพื่อจำลองการบิดคันเร่งสุดรอบเครื่องยนต์ ระบบจะคำนวณและแสดงผลกราฟแบบเรียลไทม์ผ่านกราฟแรงม้า (HP) และแรงบิด (Nm) พร้อมเอฟเฟกต์เสียงคำรามของเครื่องยนต์ที่แปรผันตามรอบเครื่องจริง

### 2. [02] AI Diagnostics HUD (หน้าจอระบบวิเคราะห์ปัญหาอัจฉริยะ)
ระบบจำลองกล่องวิเคราะห์สแกนหาสาเหตุอาการเสียของรถด้วยขุมพลัง Gemini AI:
* **OBD Live Sensor Stream (ข้อมูลเซ็นเซอร์สด)**: ตารางแสดงผลแรงดันไฟและอุณหภูมิเซ็นเซอร์ 5 ตัวแบบเรียลไทม์ ได้แก่ TPS (ลิ้นเร่ง), O2 (ไอเสีย), MAP (แรงดันไอดี), ECT (ความร้อนหม้อน้ำ), และ CKP (ความเร็วรอบ)
* **Harness Interruption (จำลองระบบไฟขาด)**: คุณสามารถกดปุ่ม **"UNPLUG"** บนเซ็นเซอร์ตัวใดก็ได้ เพื่อจำลองว่าเซ็นเซอร์ตัวนั้นเสียหรือสายไฟขาด ระบบจะสร้างรหัสโค้ดผิดปกติ (DTC) ทันที
* **AI Cyber-Scan**: กรอกอาการที่พบ หรือเลือกรหัสโค้ด DTC และกดปุ่มวิเคราะห์ ระบบจะทำการส่งข้อมูลให้ **Gemini AI** เพื่อวินิจฉัยและส่งมอบคู่มือซ่อมแซมอย่างละเอียด (Repair Protocol) เป็นภาษาไทย

### 3. [03] Manga Blueprint Specs (แคตตาล็อกชิ้นส่วนอะไหล่อะนาล็อก)
หน้าจอแผนผังสายไฟและโครงสร้างมอเตอร์ไซค์สำหรับติดตั้งพาร์ทซิ่ง:
* **Interactive Hotspots**: เลื่อนเมาส์ไปชี้ตามตำแหน่งต่าง ๆ บนมอเตอร์ไซค์เพื่อดูชื่อพาร์ทแต่ง เช่น ท่อไอเสียไทเทเนียม, หัวฉีดแรงดันสูง, เทอร์โบไฟฟ้า
* **Install/Uninstall Upgrade**: กดคลิกที่ปุ่มชิ้นส่วนเพื่อทำการติดตั้ง ชิ้นส่วนที่ติดตั้งแล้วจะส่งผลกระทบโดยตรงต่อพลังเครื่องยนต์รวม (HP เพิ่มขึ้น, แรงบิดเพิ่มขึ้น, น้ำหนักลดลง) ทันทีในแถบสถานะด้านข้างซ้าย

### 4. [04] Tuner Logs Ledger & Cyber-Invoice (ระบบบัญชีและการเงิน)
สมุดบันทึกประวัติการขึ้นไดโน่และระบบจัดการใบส่งงาน/ใบเสร็จ:
* **Tuning History**: แสดงบันทึกข้อมูลการจูนไดโน่ในแต่ละครั้งเพื่อนำมาเปรียบเทียบ
* **Work Order Controller (ระบบคำนวณใบงาน)**: ระบุชื่อลูกค้า, ชั่วโมงการทำงานช่าง (Labor Hours), อัตราค่าแรงต่อชั่วโมง (THB/HR) และเขียนบันทึกกำกับงานซ่อมแซม
* **Real-time Live Invoice**: บิลคำนวณเงินสดแสดงค่าแรงร่วมกับค่าอะไหล่แต่งที่ผู้ใช้ติดตั้งจริงแบบออโต้ พร้อมบวกภาษีมูลค่าเพิ่ม (VAT 7%)
* **Stamps & Printer (การพิมพ์ใบงาน)**: เปลี่ยนแปลงสถานะบิลได้ 3 ระดับ คือ **DRAFT** (แบบร่าง), **PENDING** (ค้างจ่าย), และ **PAID** (จ่ายเงินแล้ว - จะประทับตราอนุมัติสีเขียวเรืองแสง) พร้อมรองรับปุ่ม **"PRINT TICKET SLIP"** สำหรับสั่งพิมพ์ใบเสร็จผ่านเครื่องพิมพ์จริงหรือเซฟเป็นไฟล์ PDF

## 💻 ส่วนที่ 2: คู่มือการนำไปรันบนเซิร์ฟเวอร์ (Server Deployment Manual)

แอปพลิเคชันนี้สร้างขึ้นด้วยโครงสร้าง **Full-Stack (Vite React + Express API)** รองรับการทำงานแบบ SSR/Proxy API เพื่อปิดบังคีย์ความปลอดภัยอย่างแน่นหนา

### 📌 สิ่งที่จำเป็นต้องมีก่อนติดตั้ง (Prerequisites)
1. **Node.js**: เวอร์ชัน LTS แนะนำเวอร์ชัน 18 ขึ้นไป (เช็คโดยรัน `node -v`)
2. **NPM**: ติดตั้งมาพร้อมกับ Node.js (เช็คโดยรัน `npm -v`)
3. **Gemini API Key**: คีย์สำหรับเรียกใช้งาน AI สแกนสมองกลรถ สามารถไปสมัครเพื่อรับคีย์ฟรีได้ที่ [Google AI Studio](https://aistudio.google.com/)

### ⚙️ ขั้นตอนการตั้งค่าสภาพแวดล้อม (Environment Setup)
ให้สร้างไฟล์ชื่อ `.env` ไว้ที่โฟลเดอร์ราก (Root Directory) ของโปรเจกต์ โดยคัดลอกรูปแบบจากไฟล์ `.env.example` ไปใส่:

### 🪟 วิธีการติดตั้งและรันใช้งานบน Windows

#### ขั้นตอนที่ 1: ดาวน์โหลดโปรเจกต์และติดตั้ง Library
เปิด **Command Prompt** หรือ **PowerShell** ในโฟลเดอร์ของโปรเจกต์ จากนั้นรันคำสั่ง:
```bash
# 1. ติดตั้งไลบรารีและส่วนเสริมทั้งหมดที่โปรเจกต์ใช้งาน
npm install
```

#### ขั้นตอนที่ 2: การรันในโหมดพัฒนา (Development Mode)
ใช้คำสั่งนี้เพื่อรันเวอร์ชันพัฒนาสำหรับการทดสอบและแก้ไขโค้ด:
```bash
npm run dev
```
*ระบบจะเปิดเซิร์ฟเวอร์ที่พอร์ต **3000** สามารถเข้าใช้งานได้ทางคลังเว็บเบราว์เซอร์ที่: [http://localhost:3000](http://localhost:3000)*

#### ขั้นตอนที่ 3: การรันเพื่อใช้งานจริง (Production Build)
เมื่อระบบพร้อมสำหรับเปิดใช้งานยาวนานและมีความเสถียรสูงสุด ให้ปฏิบัติดังนี้:
```bash
# 1. รวบรวมและคอมไพล์โค้ด React กับฝั่งเซิร์ฟเวอร์ Express ให้เป็นชุดรันขนาดเล็กและรวดเร็ว
npm run build

# 2. เริ่มต้นรันเซิร์ฟเวอร์หลักที่คอมไพล์เสร็จสิ้นแล้ว (จะรันไฟล์ dist/server.cjs)
npm run start
```

### 🐧 วิธีการติดตั้งและรันใช้งานบน Linux (Ubuntu / Debian / CentOS)

การรันบนเซิร์ฟเวอร์ Linux จะแตกต่างจาก Windows เล็กน้อย โดยมีขั้นตอนการรันให้ทำงานอยู่เบื้องหลัง (Background Service) แม้เราจะปิดหน้าจอ Terminal ไปแล้ว

#### ขั้นตอนที่ 1: อัปเดตและติดตั้ง Node.js (หากยังไม่มีในเครื่อง)
```bash
# สำหรับระบบ Ubuntu/Debian
sudo apt update
sudo apt install -y nodejs npm
```

#### ขั้นตอนที่ 2: ติดตั้ง Dependencies และตั้งค่า .env
นำโค้ดโปรเจกต์เข้าไปวางในไดเรกทอรีที่ต้องการ (เช่น `/var/www/motorboy`) จากนั้นย้ายเข้าไปในโฟลเดอร์:
```bash
cd /var/www/motorboy
npm install
cp .env.example .env
nano .env  # ทำการแก้ไขใส่คีย์ GEMINI_API_KEY จริงลงไป
```

#### ขั้นตอนที่ 3: คอมไพล์โปรเจกต์
```bash
npm run build
```

#### ขั้นตอนที่ 4: การรันในเบื้องหลังถาวร (มี 2 วิธีหลักที่แนะนำ)

##### วิธีที่ 1: การใช้ PM2 (แนะนำมากที่สุด รวดเร็ว และมีระบบ Auto-Restart หากโค้ดล่ม)
1. ติดตั้งตัวจัดการโปรเซส PM2 ทั่วโลกในระบบ Linux:
   ```bash
   sudo npm install -g pm2
   ```
2. เริ่มต้นรันโปรเจกต์ MOTORBOY ด้วย PM2:
   ```bash
   pm2 start npm --name "motorboy" -- start
   ```
3. ตั้งค่าให้ PM2 เปิดรันใหม่อัตโนมัติเมื่อเซิร์ฟเวอร์บูตเครื่องใหม่:
   ```bash
   pm2 startup
   pm2 save
   ```
4. คำสั่งตรวจสอบสถานะของแอป:
   ```bash
   pm2 status      # ตรวจสอบสถานะการรัน
   pm2 logs        # ดู Log การทำงานแบบสด ๆ
   pm2 restart motorboy  # เริ่มต้นแอปใหม่เมื่อมีการอัปเดต
   ```

##### วิธีที่ 2: การสร้าง Systemd Service (วิธีมาตรฐานของระบบปฏิบัติการ Linux)
1. สร้างไฟล์คอนฟิกสำหรับบริการเซิร์ฟเวอร์:
   ```bash
   sudo nano /etc/systemd/system/motorboy.service
   ```
2. วางค่าคอนฟิกด้านล่างนี้ลงไป (ปรับเปลี่ยนตรง `/var/www/motorboy` ให้ตรงตามโฟลเดอร์ของคุณ):
   ```ini
   [Unit]
   Description=Motorboy Tokyo Override Tuning Service
   After=network.target

   [Service]
   Type=simple
   User=root
   WorkingDirectory=/var/www/motorboy
   ExecStart=/usr/bin/npm run start
   Restart=on-failure
   Environment=NODE_ENV=production

   [Install]
   WantedBy=multi-user.target
   ```
3. บันทึกไฟล์และรันคำสั่งเพื่อให้ระบบ Systemd เปิดใช้งาน:
   ```bash
   sudo systemctl daemon-reload
   sudo systemctl enable motorboy
   sudo systemctl start motorboy
   ```
4. คำสั่งจัดการบริการ:
   ```bash
   sudo systemctl status motorboy  # ตรวจเช็คสถานะการทำงาน
   sudo journalctl -u motorboy -f  # เช็คดู Log ทั้งหมด
   ```

### 🛡️ คำแนะนำเกี่ยวกับความปลอดภัยและการพอร์ตเครือข่าย
* **Port Ingress**: ตัวแอปทำงานผ่านพอร์ต **3000** หากคุณต้องการเข้าถึงแบบโดเมนทั่วไป เช่น `http://my-domain.com` โดยไม่ต้องพิมพ์ต่อท้ายด้วย `:3000` แนะนำให้ทำการติดตั้ง **Nginx** เพื่อทำเป็น Reverse Proxy ไปหาพอร์ต 3000
* **Firewall Setting**: อย่าลืมเปิดพอร์ตรับส่งข้อมูลบนไฟร์วอลล์ของคุณ:
  - บน Linux Ubuntu (UFW): `sudo ufw allow 3000/tcp`
  - บน Windows: ตรวจสอบหน้าต่าง Windows Defender Firewall เพื่ออนุญาตให้ Node.js เชื่อมต่อเครือข่ายภายนอกได้
